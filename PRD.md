# 📸 Snapshot AI - 个人碎片知识助理 PRD

## 1. 产品愿景
做一个“外挂大脑”的雏形。在未来 AI 眼镜普及前，利用手机截图作为视觉入口，帮助用户捕捉一闪而过的念头。
**核心理念：Capture Now, Process Later (现在捕捉，稍后消化)。** 将“无效的收藏”转化为“即时的知识增量”。

## 2. 核心价值主张 (Value Proposition)
*   **零摩擦输入 (Zero Friction)**：用户在刷手机时的动作必须是“无感”的。截图 -> 分享 -> 结束。绝不打断当前的浏览心流。
*   **准备好的回顾 (Ready-to-Review)**：当用户晚上打开 App 时，所有的分析结果**已经**准备就绪。无需等待 Loading，提供“拆礼物”般的即时反馈体验。
*   **增量价值**：AI 提供用户截图中**没有**但**需要**的信息（解释、拓展、行动建议）。

## 3. 用户画像与场景 (基于访谈分析)
*   **场景 A：囤积症患者 (The Hoarder)**
    *   *行为*：刷小红书/抖音时疯狂截图，心里想着“以后会看”，但永远不看。
    *   *体验*：通过“后台静默上传”，满足收集癖。晚上通过“刷卡片”的仪式感，真正消化这些内容。
*   **场景 B：即时好奇 (The Curious)**
    *   *行为*：看到一个新概念/人名，想知道是什么，但不想切 App 去搜。
    *   *体验*：截图扔进收件箱。AI 默默准备好答案，下次打开 App 时直接呈现。

## 4. 功能需求 (MVP 阶段)

### P0: 核心闭环 (The Core Loop)
1.  **无感采集 (iOS Shortcut)**
    *   **入口**：iOS 分享菜单 (Share Sheet)。
    *   **行为**：压缩图片 -> 上传至云端收件箱 (Supabase) -> 顶部通知 "已保存"。
    *   **体验**：**不跳转**到 PWA，用户停留在原 App。
2.  **云端自动处理 (Serverless Backend)**
    *   **触发器**：当图片存入 Storage 时，自动触发云端函数。
    *   **AI 分析**：云端调用 Gemini API 进行图文分析。
    *   **存储结果**：将分析好的 JSON 结构存入数据库，标记状态为 `ready`。
3.  **沉浸式回顾 (PWA)**
    *   **每日回顾模式**：打开 App，**零延迟**展示已生成好的卡片。
    *   **卡片交互**：Tinder 风格交互。
        *   **右滑 (Keep)**：保存到本地知识库（IndexedDB）。
        *   **左滑 (Trash)**：删除图片和分析结果（清理云端垃圾）。
4.  **本地知识库**
    *   保存已确认价值的卡片，供后续搜索或复习。

### P1: 体验增强
1.  **音频播放 (TTS)**
    *   针对开车/通勤场景，将“今日回顾”生成为一段 5 分钟的 Daily Podcast。
2.  **周报推送**
    *   每周日总结这一周你留下的“知识精华”。

## 5. 成功指标
*   **无感率**：从截图到恢复浏览原本内容，耗时需 < 3秒。
*   **打开即用率**：用户打开 App 后，等待内容加载的时间 < 1秒。
*   **清理率**：用户每天“清空”收件箱（无论左滑还是右滑）的比例。

# 数据标注与质量标准

## 📋 核心原则

### 🎯 增量价值原则（最重要！）

**定义**：生成的卡片内容必须提供**用户不知道的新信息**，而不是简单复述截图内容。

**判断标准**：
- ❌ Bad：只是重新排版截图中的文字
- ✅ Good：提供背景、案例、具体方法、可行动建议

**用户心理模型**：
```
用户截图 = 用户对这个话题感兴趣，但懒得深入研究
我们的价值 = 帮用户"喂到嘴边"，提供他懒得查的深度内容
```

## 📊 质量评分标准

### 维度1：增量价值（权重 50%）

| 分数 | 标准 | 示例 |
|------|------|------|
| 0分 | 完全复述截图内容，零增量 | 只列出人名和职位 |
| 5分 | 有少量背景补充，但不深入 | 只说"这个人很厉害" |
| 8分 | 提供了具体案例或方法论 | 具体的产品决策案例 |
| 10分 | 提供深度内容+可行动建议 | 案例+技术细节+如何应用 |

### 维度2：可行动性（权重 30%）

| 分数 | 标准 | 示例 |
|------|------|------|
| 0分 | 没有任何行动建议 | 只陈述事实 |
| 5分 | 有建议但太泛泛 | "可以学习一下" |
| 8分 | 有具体可执行的建议 | "问自己XXX问题" |
| 10分 | 有多层次的行动路径 | "先做X，再做Y，最后查Z" |

### 维度3：结构清晰度（权重 20%）

| 分数 | 标准 |
|------|------|
| 0分 | 混乱，无法快速理解 |
| 5分 | 有结构但不够清晰 |
| 8分 | 结构清晰，易读 |
| 10分 | 结构完美，一目了然 |

### 总分计算

```
总分 = 增量价值×0.5 + 可行动性×0.3 + 结构清晰度×0.2
```

**及格线**：总分 ≥ 7 分

## 🎯 标注流程

### 步骤1：人工标注（CSV文件）

使用 `卡片设计方案.csv` 作为标注模板：

```csv
截图文件名,卡片序号,卡片标签,卡片标题,阅读时长,核心内容摘要,我的补充
IMG_2221.PNG,1,AI工具,硅谷Builder的产品秘诀,2分钟,这6个人代表2024-2025 AI产品三个方向：速度为王、无感集成、性格设计,...,需要补充具体案例
```

### 步骤2：AI生成

运行 `python3 main.py`

### 步骤3：质量评估

对比 AI 输出和人工标注：
1. 检查是否提供增量价值
2. 打分（使用上述评分标准）
3. 记录到本文档的 Bad Cases 或 Good Cases

### 步骤4：迭代优化

- Bad Cases → 更新 Prompt
- Good Cases → 记录为示例

## 📝 Prompt 优化方向

### 当前问题

1. ❌ **缺少"增量价值"的明确要求**
   - 没有告诉AI"不要复述截图"
   - 没有要求"提供用户不知道的信息"

2. ❌ **缺少"深度挖掘"的指令**
   - 没有要求查找具体案例
   - 没有要求补充背景知识

3. ❌ **示例不够好**
   - 当前示例（年味社会学）比较抽象
   - 缺少"推荐类"内容的优质示例

### 优化方向

**核心增加**：
```
## 💡 增量价值要求（最重要！）

你的任务不是复述截图内容，而是提供**用户懒得查但需要的深度信息**。

### 具体要求：

1. **推荐类内容（书/工具/人物）**
   - ❌ 不要：只列出名字和简介
   - ✅ 要做：提供具体案例、产品哲学、可学习的方法论
   - 示例：不是"Kevin Weil是OpenAI CPO"，而是"Kevin在XX播客分享了Sora的产品决策：砍掉所有参数调节，因为..."

2. **概念解释类**
   - ❌ 不要：只解释概念本身
   - ✅ 要做：提供应用场景、现代启示、可行动建议

3. **观点类**
   - ❌ 不要：只陈述观点
   - ✅ 要做：提供论据、反例、如何应用

### 信息来源优先级：

1. **具体案例** > 抽象描述
2. **可验证的细节**（时间、地点、数据）> 泛泛而谈
3. **可行动建议** > 纯理论
4. **反常识洞察** > 常识复述
```
